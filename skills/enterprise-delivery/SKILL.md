# enterprise-delivery

## 适用场景

当任务涉及企业项目的功能交付、缺陷修复、版本发布准备、上线前核对时使用。

## 执行要求

1. 先确认需求目标、影响范围和依赖边界。
2. 优先基于已有模块扩展，不轻易引入新的抽象层。
3. 明确本次变更的验证方案，包括本地验证、联调验证或人工检查。
4. 输出内容必须包含风险点、未覆盖项和后续动作。

## 输出模板

- 变更目标
- 实现方案
- 验证方式
- 风险与限制
- 后续建议

## 边界

- 如果需求本身不清晰，应先提出澄清问题，不直接补全业务含义。
- 如果涉及权限、数据安全、合规要求，应显式提示需要额外审查。
