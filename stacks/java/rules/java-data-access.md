# Java Data Access Rules

## 目标

约束 Java 服务中的数据库访问实现，减少租户隔离、查询性能和批量写入风险。

## 规则

1. 禁止动态拼接 SQL、HQL 或原生查询，必须使用参数化语句或框架绑定参数。
2. 查询应显式选择字段、显式限制分页范围，禁止无边界 `select *`、全量扫描和大结果集下沉到应用层过滤。
3. 涉及租户、组织或数据域隔离的访问，必须在查询条件中显式体现隔离维度。
4. 批量更新、回填和清理任务必须说明批次大小、事务边界、索引命中和锁竞争风险。
5. 表结构、索引、SQL 语句和 ORM 映射调整要保持一致，不能只改其中一层。
