# Java Error Handling Rules

## 目标

统一 Java 服务中的异常语义、响应边界和日志策略，避免吞异常和用户可见错误失真。

## 规则

1. 业务异常、参数异常和系统异常要分层表达，禁止把所有失败都压成同一种运行时异常。
2. Controller 或对外接口层必须返回稳定的错误语义，不能直接暴露底层堆栈、数据库异常或框架内部信息。
3. 禁止 `catch` 后只打日志继续返回默认值，异常被吞掉时必须显式说明取舍和补偿逻辑。
4. 错误日志要保留定位上下文，例如 traceId、关键业务主键和调用路径，避免同一异常多层重复打印。
5. 面向用户的中文异常消息必须走既定国际化流程；YMS 项目按 `yms-i18n` 处理，不直接长期保留硬编码中文。
