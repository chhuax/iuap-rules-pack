# Java Backend Rules

> 这份规则在 [common/rules/core.md](../../../common/rules/core.md) 基础上补充 Java 后端约定。

## 适用范围

适用于基于 Java 的后端服务，尤其是 Spring Boot / Spring Cloud 项目。

## 规则

1. 接口、服务、仓储分层职责要清晰，不在控制器中堆积业务逻辑。
2. 异常处理应统一收口，避免直接向外暴露原始堆栈或底层错误。
3. 数据库事务边界要明确，禁止无意识扩大事务范围。
4. 新增依赖前先判断是否已有内部组件或公共库可复用。
5. 涉及并发、线程池、缓存时必须说明容量和失效策略。
