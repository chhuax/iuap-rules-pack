# Java Redis Rules

## 目标

约束 Java 服务中的 Redis 使用方式，避免阻塞命令、缓存雪崩和不安全的分布式锁实现。

## 规则

1. 禁止在业务链路中使用 `keys *`、`flushdb`、`flushall`、`monitor` 等高风险命令。
2. Key 命名必须体现业务前缀和隔离维度；涉及租户数据时，Key 中要显式带租户信息。
3. 缓存数据默认要设置过期时间，并通过抖动或分散过期避免同一时间集中失效。
4. 大集合遍历必须使用 `scan`/`hscan`/`sscan` 等渐进式方式，禁止对大 Key 直接做全量读取。
5. 分布式锁必须包含过期时间、持有者标识和释放保护，不能只依赖 `setnx` 或无超时锁。
