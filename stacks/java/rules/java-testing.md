# Java Testing Rules

## 目标

统一 Java 服务的最小测试要求，避免高风险改动缺少自动化验证或依赖脆弱测试。

## 规则

1. 核心业务、异常路径、边界条件和回归修复必须有与风险匹配的自动化验证，缺失时要显式说明原因。
2. 默认优先单元测试和切片测试，只有确实需要跨层验证时才启动完整 Spring 上下文。
3. 测试应断言可观察行为，不要只验证内部实现细节或无业务意义的调用顺序。
4. 禁止依赖 `sleep`、固定时序或环境偶然性构建脆弱测试；并发和异步测试要有确定结束条件。
5. 测试数据、脚本和断言要保持可重复运行，不把本地环境、人工顺序或历史脏数据当成前提。
