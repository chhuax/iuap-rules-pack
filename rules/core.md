# IUAP Core Rules

## 目标

统一团队在需求实现、代码变更、评审、验证与交付阶段的最小执行标准。

## 基础原则

1. 先澄清边界，再开始实现，避免带着模糊需求直接编码。
2. 变更必须可验证，至少提供与改动风险相匹配的检查方式。
3. 不允许静默绕过错误、测试或校验，必须显式说明取舍。
4. 修改现有代码时优先遵循已有结构和约定，避免无必要重构。
5. 涉及接口、数据、安全、权限的改动必须补充影响说明。

## 交付要求

- 提交说明应明确改动目的、范围和验证方式。
- 评审结论优先关注回归风险、兼容性风险和遗漏验证。
- 生产相关变更应记录发布前检查项与回滚思路。
